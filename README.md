- **リモート追跡ブランチについて**
Gitを使うことでローカルリポジトリからリモートリポジトリにアクセスできるようになりますが、
リモートのブランチをローカルから参照するために使われるのがリモート追跡ブランチです。
これは`git branch -r`で表示されるブランチのことですが、それぞれ`<リモート名>/<リモートのブランチ名>`という意味です。
origin/masterのことをリモート追跡ブランチであると説明する記事もあるようですが、
originとmasterというのは決して特別な名前でもないということに注意が必要だそうです。
> “origin” は特別なものではない
Git の “master” ブランチがその他のブランチと何ら変わらないものであるのと同様に、 “origin” もその他のサーバーと何ら変わりはありません。 “master” ブランチがよく使われている理由は、ただ単に git init がデフォルトで作るブランチ名がそうだからというだけのことでした。 同様に “origin” も、git clone を実行するときのデフォルトのリモート名です。 たとえば git clone -o booyah などと実行すると、デフォルトのリモートブランチは booyah/master になります。

参照: [3.5 Git のブランチ機能 - リモートブランチ](https://git-scm.com/book/ja/v2/Git-%E3%81%AE%E3%83%96%E3%83%A9%E3%83%B3%E3%83%81%E6%A9%9F%E8%83%BD-%E3%83%AA%E3%83%A2%E3%83%BC%E3%83%88%E3%83%96%E3%83%A9%E3%83%B3%E3%83%81)

#terminalLastCommand

# 条件
- あなたはRuby on Railsのエキスパートです。

- 必ず**どのファイルの何行目でどんなエラーが出ているか**を説明してください。
- 該当エラーの解決策を提示してください。
- 必ず**RSpecのテストファイルを修正**してください。
- エラーを解決するためには、**どのファイルを修正すればいいのか**説明してください。
- ファイル名は**現在いる作業ディレクトリからの相対パス**で示してください。

# 出力フォーマット

## 1. エラー文の提示と和訳
### エラー文
### 和訳
## 2. エラーの説明

### 説明

```
Error response from daemon: NotFound: content digest sha256:***: not found
```
サポーターの方に画面共有で教えてもらっても解決しなかったので、container, image, volume, builds全て削除してデプロイ資料をもとに一から始めることになりました。（こういう時結構しんどいですね🥲

前回ポート番号の被りが原因で止まった時は`compose.yml`でポート番号いじればそれで解決だったのですが、今回は全く別のエラーでエラー文読んでも理解できない系に遭遇したので焦りました、、
